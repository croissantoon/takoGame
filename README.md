## 今のシステム
プレイヤーはスキルを選んで行動する。スキルはSPを使う。\
攻撃スキルは相手にダメージと衝撃を与える。(HPとStaggerを削る)\
Staggerという値は０になると一時的に行動不能になる。\
その間に攻撃を受けたり１ターン経ったりするとStaggerは全回復し、再び行動可能になる。\
行動不能の間は被ダメージが防御貫通*２倍のダメージとなる。\
プレイヤーの行動の後にエネミーが行動する。\
敵を倒しても何もおこらない←どうにかしろよ

## 将来的には...
プレイヤーはスキルカードを持つようになる。\
手札と山札があり、手札にあるカードを使うと、山札の一番下に置かれ、一番上から手札に置かれる。\
スキルカードは装備品によって決めることができる。スキルカードは最大20枚まで山札に置くことができる。\
敵を倒すと装備品やアイテムを落とすようになる。（ハクスラみたいな感じ）

# update
## 2025/11/04
### 各スキルのクラス化
1つのメソッドにswitchとcaseを使って複数のスキルを入れるのではなく、Skillというinterfaceを作り、それからimplementされたクラスを作りました。\
それぞれのクラスはMainクラスにてリストで管理され、プレイヤーが入力した番号のスキルを発動します。\
各スキルのクラスはSillからimplementされているので、同じ形態を持つので、編集しやすくなるはずです。\
### スキルの選択方法の変更
Mainクラスのuseメソッドを廃止し、代わりに各クラスからメソッドを取り出します。\
スキルはスキルカードとなり、山札と手札に分けられます。\
山札にはシャッフルされたカードがいくつかあります。\
プレイヤーがカードを使うと山札の1番下に戻る、というシステムを実現しました。
